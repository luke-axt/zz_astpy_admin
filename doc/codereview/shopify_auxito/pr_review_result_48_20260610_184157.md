### 🟠 严重（推荐修复）

#### 1. Cart Drawer 图片延迟加载导致首屏体验降级

- **位置**: `snippets/atc-banner.liquid:24`, `snippets/atc-banner.liquid:4727`, `snippets/atc-banner.liquid:4878`, `snippets/atc-banner.liquid:4900`, `snippets/cart-discount-panel.liquid:14`, `sections/auxito-contact-page.liquid:3492`, `sections/auxito-search-page.liquid:4229` 等
- **问题**: 大量关键图片（cart logo、confidence icons、checkout lock icon、discount tag icon、contact/search page icons）均标记了 `loading="lazy"`。Cart Drawer 是用户点击 Add-to-Cart 后立即展示的交互组件，其中 logo、产品图、支付图标、信任图标均为首屏必现内容。`loading="lazy"` 在部分浏览器中会导致图片延迟加载，造成抽屉打开后数秒内出现空白/灰块，严重影响购物体验。
- **影响**: 购物车抽屉打开时图片闪烁或延迟出现，降低转化率和用户信任感。
- **修复方案**: Cart Drawer 内的关键图片应移除 `loading="lazy"` 或改为 `loading="eager"`。例如：
  ```liquid
  <!-- 建议修改 -->
  <img class="atc-cart-logo" src="{{ auxito_cart_logo }}" alt="AUXITO" width="210" height="42">
  ```
  对于联系/搜索页面中视口内的 hero 图片和卡片图标，也应评估是否需要 `loading="lazy"`。

#### 2. `data-item-key` 回退逻辑存在空字符串风险

- **位置**: `assets/auxito-cart-drawer.js:908`, `assets/auxito-cart-drawer.js:934`
- **问题**: `updateItemQuantity` 和 `removeItem` 方法使用 `$(input).attr("data-item-key") || $(input).attr("data-product_id")` 获取 item ID。若 `data-item-key` 存在但值为空字符串 `""`，`||` 运算符不会回退到 `data-product_id`，导致后续 AJAX 请求发送空 ID。
- **影响**: 用户更新数量或删除商品时可能静默失败，或错误地操作了其他商品。
- **修复方案**: 使用更严格的空值检查：
  ```js
  var itemId = $(input).attr("data-item-key") || $(input).attr("data-product_id");
  // 建议改为：
  var rawKey = $(input).attr("data-item-key");
  var itemId = (rawKey && String(rawKey).trim()) ? rawKey : $(input).attr("data-product_id");
  ```

#### 3. 联系表单内联脚本缺少错误恢复与网络超时处理

- **位置**: `sections/auxito-contact-page.liquid:3593-3633`
- **问题**: 内联脚本在表单提交时设置 `button.disabled = true` 和 `button.classList.add('is-loading')`，但没有处理以下场景：
  1. 网络错误导致表单提交失败（浏览器会回退到普通提交，但按钮状态仍被锁定）；
  2. 表单验证通过但服务器返回 500/502 错误；
  3. 用户点击浏览器返回按钮后，按钮仍显示 "SENDING..." 状态。
- **影响**: 极端情况下按钮被永久锁定，用户无法再次提交。
- **修复方案**: 
  1. 监听 `pageshow` 事件，在页面从 bfcache 恢复时重置按钮状态；
  2. 添加 `window.onbeforeunload` 保护或在表单提交失败时通过 `setTimeout` 自动重置按钮状态（例如 15 秒超时）。
  ```js
  window.addEventListener('pageshow', function () {
    var btn = form.querySelector('[data-auxito-contact-submit]');
    if (btn) {
      btn.disabled = false;
      btn.classList.remove('is-loading');
      btn.removeAttribute('aria-busy');
      var label = btn.querySelector('[data-auxito-contact-submit-label]');
      if (label && btn.dataset.originalLabel) label.textContent = btn.dataset.originalLabel;
      form.dataset.auxitoSubmitting = '';
    }
  });
  ```

#### 4. CSS 过度使用 `!important` 降低可维护性

- **位置**: `snippets/atc-banner.liquid`（新增 500+ 行 CSS）、`assets/auxito-support-pages.css`（全文件）
- **问题**: Cart Drawer 的样式定义中大量使用了 `!important`（例如 `.atc-banner--container.auxito-cart-drawer` 下的几乎所有属性）。这使得后续通过 Theme Editor 或自定义 CSS 覆盖样式变得极为困难，也增加了调试成本。
- **影响**: 长期维护困难；商家或开发者在后台自定义样式时可能无法生效。
- **修复方案**: 
  1. 优先通过提高选择器特异性来覆盖样式，而非 `!important`；
  2. 对于必须保护的样式（如 drawer 的尺寸和布局），保留必要的 `!important`，但减少无差别使用。

### 🟡 一般（建议修复）

#### 5. 硬编码 CDN URL 缺乏版本控制与容错机制

- **位置**: `sections/auxito-contact-page.liquid:3309-3333`、`sections/auxito-search-page.liquid:4129-4146`、`snippets/atc-banner.liquid:4713-4718`、`snippets/cart-discount-panel.liquid:5`
- **问题**: 大量图片资源使用硬编码的 Shopify CDN URL（如 `https://cdn.shopify.com/s/files/1/0012/8063/9049/files/auxito-support-contact-hero-v2.jpg?v=1780984561`）。这些 URL 的 `v` 参数是缓存版本号，如果 CDN 上的文件被替换或删除，链接会失效。此外，代码中没有 fallback 逻辑。
- **影响**: CDN 资源失效时页面出现空白或 404。
- **修复方案**: 
  1. 将关键图片资源纳入主题 `assets/` 目录管理，使用 `asset_url` 过滤器；
  2. 若必须使用外部 CDN，建议在 `settings_schema.json` 中定义为全局设置，方便运营人员更新；
  3. 为关键图片添加 `onerror` 回退或默认占位图。

#### 6. 搜索页面品牌列表硬编码，不利于维护

- **位置**: `sections/auxito-search-page.liquid:4127`
- **问题**: `default_bulb_guide_brands` 变量中硬编码了 41 个汽车品牌。虽然可通过 section settings 覆盖，但默认值维护成本高，且缺少国际化支持。
- **影响**: 新增或删除品牌时需要修改代码并重新部署主题。
- **修复方案**: 
  1. 考虑将品牌列表存储在 `settings_data.json` 或 Shopify Metaobject 中；
  2. 或通过动态导航菜单（Linklists）获取品牌列表，实现无代码更新。

#### 7. `stopImmediatePropagation` 可能破坏第三方插件的事件监听

- **位置**: `sections/product-collection.liquid:101`、`assets/search-result-quickshop.js:3231`
- **问题**: 在 Quick Buy 按钮的点击事件处理中新增了 `event.stopImmediatePropagation()`。这会阻止同一元素上其他事件监听器的执行，可能导致 Judge.me、Gorgias Chat 或其他第三方应用的事件回调失效。
- **影响**: 第三方功能（如点击统计、聊天窗口触发）可能在产品卡片上失效。
- **修复方案**: 
  1. 确认是否需要 `stopImmediatePropagation`，或仅用 `stopPropagation` 阻止冒泡即可；
  2. 若必须使用，应在注释中说明原因，并测试与现有第三方脚本的兼容性。

#### 8. `templates/page.car-guide.liquid` 与 `page.guide.liquid` 字段名变更的兼容性

- **位置**: `templates/page.car-guide.liquid:115`、`templates/page.guide.liquid:115`
- **问题**: `name="contact[message]"` 被改为 `name="contact[body]"`。Shopify 的标准联系表单使用 `contact[body]`，这是正确的修复。但需确认是否有自定义的 Email 通知模板或第三方应用（如 CRM 集成）依赖于旧字段名 `contact[message]`。
- **影响**: 若后端邮件模板未同步更新，可能导致客户留言内容为空。
- **修复方案**: 部署前检查 Shopify Admin > Settings > Notifications 中的 "Contact customer" 邮件模板是否使用 `{{ body }}` 变量。

#### 9. `sections/auxito-search-page.liquid` 缺少 `submodel` 支持

- **位置**: `sections/auxito-search-page.liquid:4185-4223`
- **问题**: 车辆选择器只实现了 `year`、`make`、`model` 三个字段，但 AUXITO 的主题中其他车辆选择器（如首页）支持 `submodel`。搜索页面的选择器缺少这一字段，可能导致部分车型的搜索结果不准确。
- **影响**: 用户选择了 Year/Make/Model 后，如果该车型有多个 Sub-model，可能返回不匹配的产品。
- **修复方案**: 评估是否需要在搜索页面车辆选择器中添加 Sub-model 步骤，或至少在产品页面提供 Sub-model 筛选。

#### 10. `layout/theme.liquid` 中 Judge.me 浮动标签隐藏逻辑过于宽泛

- **位置**: `layout/theme.liquid:3268`
- **问题**: 代码通过 `is_auxito_support_route` 变量在联系、搜索、touch 页面隐藏 Judge.me floating tab。这是通过 `render_static_vehicle_in_layout or is_auxito_support_route` 条件实现的。若未来新增支持页面，需要同步更新此条件。
- **影响**: 维护成本略高，容易遗漏。
- **修复方案**: 考虑在 `is_auxito_support_route` 的定义处添加注释，提醒维护者同步更新隐藏逻辑。

#### 11. `snippets/navmenu.liquid` 移除 mega menu 图片预加载可能影响首次展开体验

- **位置**: `snippets/navmenu.liquid:1087`
- **问题**: `scheduleMegaImagePreload()` 被注释掉，注释说明是为了避免首屏加载时预加载隐藏的图片。这是性能优化，但可能导致用户首次展开 mega menu 时图片加载延迟。
- **影响**: 首次展开 mega menu 时可能出现图片闪烁。
- **修复方案**: 建议改为在用户鼠标 hover 到导航项时才开始预加载对应 mega menu 的图片，而非完全移除预加载逻辑。

## 审查发现

### 🟡 一般（建议修复）

#### 1. `box_weight_tmp` 空字符串/纯空格未归一化为 `None`，可能导致数据库写入非预期值

- **位置**：`ETL/ods/Logistic/ImportInputLgtFeeCheckInfo.py:177-180`
- **问题**：
  ```python
  if row.get('box_weight_tmp') is not None and str(row['box_weight_tmp']).strip() != '':
      check_field_type(row['box_weight_tmp'], '实重-草稿')
  ```
  该逻辑仅跳过校验，但未将空字符串/纯空格归一化为 `None`。若 Excel 读取结果为 `''` 或 `'   '`，该值会被原样带入 `data_list` 并写入 MySQL。
- **影响**：
  - MySQL `strict_mode` 下，`''` 写入 `DECIMAL` 列会触发 `ERROR 1366 (HY000): Incorrect decimal value`，导致整批数据插入失败。
  - 非严格模式下，`''` 会被静默转为 `0.00`，与业务语义“空值/未填写”不符，造成数据污染。
- **修复方案**：
  与 `total_calc_weight_wh` 的处理保持一致，在校验前先进行归一化：
  ```python
  # 归一化 box_weight_tmp：空白字符串统一转为 None
  if isinstance(row.get('box_weight_tmp'), str) and row['box_weight_tmp'].strip() == '':
      row['box_weight_tmp'] = None

  if row.get('box_weight_tmp') is not None:
      check_field_type(row['box_weight_tmp'], '实重-草稿')
  ```

#### 2. 缺少针对本次改动的单元测试/集成测试

- **位置**：本次 PR 无测试文件变更
- **问题**：两项行为变更（空白字符串归一化、允许空值跳过校验）均未在测试层覆盖。
- **影响**：后续重构或调整校验逻辑时，无法快速回归验证；潜在边界条件（如 `NaN`、`0`、`'0'`、`'  '`）缺乏自动化防护。
- **修复方案**：
  补充至少以下场景的测试用例：
  - `total_calc_weight_wh` 为 `''`、`'  '`、`None`、`0`、`'1.5'` 时的归一化结果。
  - `box_weight_tmp` 为 `None`、`''`、`'  '`、`0`、`'1.5'` 时是否通过校验且最终值正确。

### 🟢 优化（可选）

#### 3. `total_calc_weight_wh` 未覆盖 pandas / openpyxl 可能产生的 `NaN`

- **位置**：`ETL/ods/Logistic/ImportInputLgtFeeCheckInfo.py:137-139`
- **问题**：当前仅判断 `isinstance(..., str)`。若上游使用 pandas 读取 Excel，空单元格可能为 `float('nan')` 或 `pandas.NaT`，此时 `isinstance(NaN, str)` 为 `False`，`NaN` 会被直接写入 MySQL，部分驱动下同样会导致 `decimal` 列报错或存储异常。
- **修复方案**：
  引入通用空值判断（如 `pandas.isna()` 或自定义 `is_blank(value)`），或在该字段也做一次 `try/except float()` 兜底转换。

#### 4. 两处清洗逻辑风格不一致，建议抽取为通用函数

- **位置**：同上
- **问题**：`total_calc_weight_wh` 使用 `isinstance + strip`，`box_weight_tmp` 使用 `get + str + strip`，逻辑分散且语义重复。
- **修复方案**：
  在模块内增加一个小型工具函数：
  ```python
  def normalize_blank_to_none(row, key):
      val = row.get(key)
      if isinstance(val, str) and val.strip() == '':
          row[key] = None
  ```
  两处统一调用，提升可维护性。

#### 5. `check_field_type` 中使用裸 `except:`（前置遗留）

- **位置**：`ETL/ods/Logistic/ImportInputLgtFeeCheckInfo.py:155-159`
- **问题**：
  ```python
  try:
      float(field_value)
  except:
      flag = False
  ```
  裸 `except` 会捕获 `KeyboardInterrupt`、`SystemExit` 等，不符合 Python 最佳实践。
- **修复方案**：
  改为 `except (TypeError, ValueError):`，范围更精确，避免意外吞掉系统异常。

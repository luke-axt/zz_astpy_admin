## 🔴 高危（必须修复）

### 1. `refresh_bal_data` 方法成功时无返回值，导致 `action` 崩溃
- **位置：** `dwd/adl/ebay/EbaySuspectOrderMonitorJob.py:67-76`
- **问题：** 方法仅在 `res.is_fail()` 时 `return res`，成功分支没有显式返回，隐式返回 `None`。
- **影响：** `action` 中 `res = self.refresh_bal_data(count_day)` 后执行 `res.is_fail()` 会抛出 `AttributeError: 'NoneType' object has no attribute 'is_fail'`，**Job 直接崩溃**。
- **修复方案：** 方法末尾添加 `return ResultObj.success()`。

### 2. `run_adl_push_data` 方法成功时无返回值，导致 `action` 崩溃
- **位置：** `dwd/adl/ebay/EbaySuspectOrderMonitorJob.py:78-87`
- **问题：** 与上一条完全相同，成功分支无返回值。
- **影响：** `action` 中 `res = self.run_adl_push_data()` 后调用 `res.is_fail()` 同样崩溃。
- **修复方案：** 方法末尾添加 `return ResultObj.success()`。

## 🟠 严重（推荐修复）

### 3. 企业微信敏感信息（Token 和 AgentID）硬编码
- **位置：** `dwd/adl/ebay/EbaySuspectOrderMonitorJob.py:137-138, 147`
- **问题：** `qywx_add_token="BFj9jQo4t4OV1Ni8bNWXxRNLfDi"` 和 `agentid: 1000022` 直接硬编码在源码中。
- **影响：** 敏感信息泄露风险；Token 变更需改源码重新部署。
- **修复方案：** 抽取到配置文件或 `adminconfig` 中读取。

### 4. `send_qywx_message` 异常处理使用裸 `except` 且异常时不重试
- **位置：** `dwd/adl/ebay/EbaySuspectOrderMonitorJob.py:131-155`
- **问题：** 裸 `except:` 捕获所有异常；`except` 块内无 `continue`，异常时直接退出重试循环。
- **影响：** 网络抖动时直接失败，重试机制失效。
- **修复方案：** 改为 `except Exception:`，并添加 `time.sleep(retry_interval)` + `continue`。
